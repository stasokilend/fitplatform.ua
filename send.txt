Виконай роль Senior Full Stack розробника та архітектора програмного забезпечення.

Необхідно створити основу дипломного проєкту на тему:

«Розробка вебплатформи для персоналізованих фітнес-тренувань та моніторингу здоров'я».

Проєкт створюється для кваліфікаційної роботи бакалавра спеціальності 126 «Інформаційні системи та технології».

Головна мета: створити реальний робочий вебзастосунок, який потім буде використовуватися як практична частина дипломної роботи.

Технології:

- PHP 8+
- HTML5
- CSS3
- JavaScript
- MySQL
- OpenServer як локальне середовище розробки

Архітектура має бути максимально простою, зрозумілою та придатною для опису в дипломній роботі.

======================

ФУНКЦІОНАЛ ПЛАТФОРМИ

Система повинна містити:

1. Головну сторінку.

На головній сторінці необхідно розмістити:

- короткий опис платформи;
- переваги;
- кнопки «Увійти» та «Зареєструватися»;
- адаптивний сучасний дизайн.

2. Реєстрацію користувача.

Поля:

- ім'я;
- електронна пошта;
- пароль.

3. Авторизацію користувача.

4. Особистий кабінет користувача.

У кабінеті необхідно додати:

Особисті дані:

- стать;
- вік;
- зріст;
- вага;
- рівень фізичної підготовки;
- мета тренувань.

Мета тренувань:

- схуднення;
- набір маси;
- підтримка форми;
- покращення витривалості.

5. Генерацію персонального плану тренувань.

Система повинна автоматично формувати рекомендації залежно від введених параметрів.

Поки що використовувати просту логіку через умови if/else.

Наприклад:

Якщо:

мета = схуднення

то:

- кардіо;
- ходьба;
- біг;
- вправи на все тіло.

Якщо:

мета = набір маси

то:

- силові вправи;
- тренування на групи м'язів.

6. Моніторинг прогресу.

Необхідно реалізувати:

- додавання поточної ваги;
- відображення історії змін;
- простий графік прогресу за допомогою JavaScript.

7. Інформаційну панель.

Відобразити:

- кількість тренувань;
- поточну вагу;
- поставлену ціль;
- останні результати.

ВИМОГИ ДО КОДУ

1. Показуй повністю код кожного файлу.

2. Перед кожним файлом обов'язково пиши його шлях.

Наприклад:

Файл: config/database.php

Потім повний код.

3. Не пропускай жодного файлу.

4. Не використовуй псевдокод.

5. Не пиши:

«тут додайте самостійно»

«реалізуйте самі»

«цей код можна дописати пізніше»

Усе повинно працювати одразу після створення файлів.

6. Після завершення кожного великого блоку пояснюй коротко його призначення.

НЕ описуй кожну змінну.
